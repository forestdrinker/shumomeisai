# 数据文件对比报告

## 1. 文件概览

本报告对比了以下两个数据文件的内容与结构：
1.  **CSV 文件**: `2026_MCM_Problem_C_Data.csv`
2.  **Excel 文件**: `DWTS_All_Seasons_1-34.xlsx`

### 结论摘要
*   **覆盖范围一致**：两个文件均完整覆盖了 **第1季 至 第34季** 的数据。
*   **详细度不同**：CSV 文件包含详细的比赛数据（每周得分、排名、裁判评分），适合深入分析；Excel 文件仅为赛季维度的前三名摘要。
*   **一致性极高**：经核对，所有赛季的冠军归属在两个文件中完全一致（仅存在细微的字符编码差异，如 `Hélio` vs `Helio`）。

---

## 2. 结构对比

| 特征             | CSV (`2026_MCM_Problem_C_Data.csv`)       | Excel (`DWTS_All_Seasons_1-34.xlsx`) |
| :--------------- | :---------------------------------------- | :----------------------------------- |
| **数据粒度**     | **选手/组合级别** (每行代表一个参赛组合)  | **赛季级别** (每行代表一个赛季)      |
| **行数 (Shape)** | **421 行** (所有参赛组合)                 | **34 行** (34个赛季)                 |
| **列数**         | **53 列**                                 | **4 列**                             |
| **主要内容**     | 选手信息、最终排名、第1-11周裁判详细评分  | 赛季、冠军、亚军、季军名单           |
| **用途建议**     | **核心建模与分析** (裁判偏好、得分趋势等) | **快速查询** (历届前三名)            |

### CSV 文件关键列：
*   `celebrity_name` (名流姓名), `ballroom_partner` (舞伴)
*   `season` (赛季)
*   `results` (结果), `placement` (最终排名)
*   `week1_judge1_score` ... `week11_judge4_score` (每周裁判评分)

### Excel 文件关键列：
*   `Season` (赛季标识，如 S1)
*   `Winner` (冠军组合)
*   `Runner-up` (亚军组合)
*   `Third Place` (季军组合)

---

## 3. 内容一致性验证

我们对比了 Excel 中记录的冠军与 CSV 中 `placement = 1` 的记录，结果如下：

*   **赛季覆盖**：两个文件均包含 Season 1 到 Season 34，无缺失赛季。
*   **冠军一致性**：34个赛季的冠军名单完全匹配。
    *   *注*：第5季存在拼写差异，Excel 为 `Hélio`，CSV 为 `Helio Castroneves`，确认为同一人。

## 4. 建议

如果您需要进行 **数学建模**（如题目C要求的评分模型分析），请**主要使用 CSV 文件** (`2026_MCM_Problem_C_Data.csv`)。它提供了每周的详细得分数据，这是构建模型和分析评分偏差的基础。Excel 文件目前仅作为结果核对的参考即可。
